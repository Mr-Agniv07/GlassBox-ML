Linear Regression

Fit a straight line to data by gradient descent — watch it learn, step by step. No libraries. The math below is the math running on screen.

🧠 Intuition

We have a cloud of points and we believe a straight line ŷ = w·x + b roughly explains them. The line starts flat and wrong. We measure how wrong (the error), figure out which way to nudge the slope w and intercept b to make it less wrong, and take a tiny step that way. Repeat a few hundred times and the line settles into the best fit. That nudge-toward-less-wrong is gradient descent — the engine behind almost everything in ML, including XGBoost later.

📐 The math

Prediction for a point:

ŷᵢ = w·xᵢ + b

How wrong we are overall — Mean Squared Error:

MSE = (1/n) · Σ (ŷᵢ − yᵢ)²

Which way to nudge each knob (the gradients — slope of the error w.r.t. each knob):

∂MSE/∂w = (2/n) · Σ (ŷᵢ − yᵢ)·xᵢ     ∂MSE/∂b = (2/n) · Σ (ŷᵢ − yᵢ)

The update — step downhill by a small learning rate η:

w ← w − η·(∂MSE/∂w)     b ← b − η·(∂MSE/∂b)

⌨️ The code — this exact function is what runs above

// One step of gradient descent — pure JS, no libraries.
function gradientStep(points, w, b, lr) {
  const n = points.length;
  let dw = 0, db = 0, sse = 0;
  for (const {x, y} of points) {
    const err = (w * x + b) - y;   // how wrong on this point
    dw  += err * x;                // ∂MSE/∂w  (before the 2/n)
    db  += err;                    // ∂MSE/∂b
    sse += err * err;              // for reporting MSE
  }
  dw = (2 / n) * dw;
  db = (2 / n) * db;
  return {
    w:   w - lr * dw,              // step the slope downhill
    b:   b - lr * db,              // step the intercept downhill
    mse: sse / n
  };
}
