k-Means Clustering

No labels at all — the algorithm finds the groups itself. Watch centroids glide to the centers of their clusters, one assign-update step at a time. Your first unsupervised algorithm.

🧠 Intuition

Drop k centroids at random, then repeat two steps until nothing moves: assign — colour every point by its nearest centroid; then update — slide each centroid to the average of the points that chose it. Each round can only lower the total within-cluster distance (the inertia), so it always converges — though to a local optimum that depends on the random start.

📐 The math

Assign — each point takes the label of its nearest centroid:

labelᵢ = argminc ‖ xᵢ − μc ‖²

Update — each centroid becomes the mean of its assigned points:

μc = (1 / |Cᵧ|) · Σxᵢ ∈ Cᵧ xᵢ

Both steps minimise the same objective (inertia), which only ever goes down:

J = Σᵢ ‖ xᵢ − μlabelᵢ ‖²

⌨️ The code — the two steps that run above

// One k-means iteration: assign, then update.
function kmeansStep(points, centroids) {
  // 1. ASSIGN: each point picks its nearest centroid
  const labels = points.map(p => {
    let best = 0, bestD = Infinity;
    centroids.forEach((c, j) => {
      const d = (p.x - c.x)**2 + (p.y - c.y)**2;
      if (d < bestD) { bestD = d; best = j; }
    });
    return best;
  });

  // 2. UPDATE: each centroid moves to the mean of its points
  const next = centroids.map((c, j) => {
    const pts = points.filter((_, i) => labels[i] === j);
    if (pts.length === 0) return c;                 // empty cluster: stay put
    const mx = pts.reduce((s, p) => s + p.x, 0) / pts.length;
    const my = pts.reduce((s, p) => s + p.y, 0) / pts.length;
    return { x: mx, y: my };
  });

  return { labels, next };
}

Unsupervised: notice points never carries a label — the algorithm invents the groups.
