k-Nearest Neighbors

No training at all — just remember every point, then classify a new one by asking its k closest neighbors to vote. Move the query around and watch the neighbors light up.

🧠 Intuition

KNN is lazy: there's no model to fit. To classify a new point, measure how far it is from every stored point, pick the k nearest, and take a majority vote of their labels. Small k = jagged, sensitive boundaries (low bias, high variance); large k = smooth, blurry boundaries (high bias, low variance). The background shading shows how the entire plane would be classified.

📐 The math

Distance from the query x to each stored point xᵢ (Euclidean):

d(x, xᵢ) = √ Σⱼ (xⱼ − xᵢⱼ)²

Sort by distance, keep the k nearest, and predict the most common label among them:

ŷ = mode( { yᵢ : xᵢ ∈ k nearest neighbours of x } )

That's the whole algorithm — no weights, no loss, no gradient. The cost is all at query time.

⌨️ The code — this exact function is what runs above

// Classify a query point by majority vote of its k nearest neighbours.
function knnPredict(points, qx, qy, k) {
  // distance from the query to every stored point
  const d = points.map((p, i) => ({
    i, dist: Math.hypot(p.x1 - qx, p.x2 - qy)
  }));
  d.sort((a, b) => a.dist - b.dist);          // nearest first
  const idx = d.slice(0, k).map(o => o.i);    // the k nearest

  const votes = [0, 0, 0];                    // tally their labels
  idx.forEach(i => votes[points[i].y]++);

  let label = 0;                              // the winner
  for (let c = 1; c < votes.length; c++)
    if (votes[c] > votes[label]) label = c;

  return { label, idx, votes };
}

No fit() loop anywhere — the work happens entirely at query time.
