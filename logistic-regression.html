Logistic Regression

Draw a boundary between two classes by gradient descent — watch the boundary rotate into place and the confidence sharpen. No libraries. The math below is the math running on screen.

🧠 Intuition

We have two kinds of points — blue (class 0) and red (class 1) — and we want a line that separates them. But instead of a hard line, logistic regression outputs a probability: squash the linear score z = w·x + b through the sigmoid so it lands in (0, 1). Points far on one side → near 0, far on the other → near 1, and right on the boundary → 0.5. We nudge w and b to make those probabilities match the true labels. The p = 0.5 contour is the decision boundary.

📐 The math

Linear score, then squash to a probability:

z = w·x + b     p = σ(z) = 1 / (1 + e−z)

How wrong we are — log-loss (binary cross-entropy):

L = −(1/n) · Σ [ yᵢ·log(pᵢ) + (1−yᵢ)·log(1−pᵢ) ]

The gradients — identical in form to linear regression (that's the magic of GLMs):

∂L/∂w = (1/n) · Σ (pᵢ − yᵢ)·xᵢ     ∂L/∂b = (1/n) · Σ (pᵢ − yᵢ)

Same downhill step as before:

w ← w − η·(∂L/∂w)     b ← b − η·(∂L/∂b)

⌨️ The code — this exact function is what runs above

// One step of gradient descent for logistic regression — pure JS.
function gradientStep(points, w, b, lr) {
  const n = points.length;
  let dw0 = 0, dw1 = 0, db = 0, loss = 0;
  for (const {x1, x2, y} of points) {
    const z = w[0]*x1 + w[1]*x2 + b;
    const p = 1 / (1 + Math.exp(-z));      // sigmoid -> probability
    const err = p - y;                     // SAME error term as linear reg
    dw0 += err * x1;
    dw1 += err * x2;
    db  += err;
    const eps = 1e-12;                      // keep log() finite
    loss += -(y*Math.log(p+eps) + (1-y)*Math.log(1-p+eps));
  }
  return {
    w: [w[0] - lr*dw0/n, w[1] - lr*dw1/n], // step the boundary
    b: b - lr*db/n,
    loss: loss / n
  };
}

Note the error term p − y is identical to linear regression's — only the prediction changed.
